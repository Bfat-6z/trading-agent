**Headline verdict: the architecture is right.** LLM-proposes → deterministic-validates → mechanical-fires is the only empirically defensible LLM role (survey: LLMs are "supervised research assistants, not autonomous decision-makers", https://arxiv.org/pdf/2510.05533); our lockbox+novelty+bootstrap is exactly what the decay/data-snooping literature prescribes. ## 1) TOP-10 IMPROVEMENTS (ranked)

**#1 — Re-parameterize exit grid in ATR/σ units, not raw %.** (Effort: M)
Current: `sl_pct`/`tp_pct` raw % (rounded 0.01), grid 16 SL×TP×TO combos per coin. This silently re-optimizes per vol regime and doesn't transfer across coins. Literature: triple-barrier with vol-scaled horizontal barriers TP=P0(1+τp·σ), SL=P0(1−τs·σ) (https://alm0stsurely.github.io/2026/03/05/triple-barrier-labeling/, https://www.newsletter.quantreo.com/p/the-triple-barrier-labeling-of-marco). Highest payoff, zero new machinery — new grid enters as trials through the normal pipeline. Requires DSL v3 fields (`sl_atr`/`tp_atr`) + hash bump.

**#2 — Purge + embargo at train/OOS/lockbox boundaries.** (Effort: S)
Labels span up to `timeout` bars (default 16 = 4h); contiguous chronological splits leak overlapping label windows at 15m frequency. Drop ≥`timeout` bars at each boundary (embargo ~1 day). Block bootstrap does NOT fix train/test leakage (https://en.wikipedia.org/wiki/Purged_cross-validation, https://www.sciencedirect.com/science/article/abs/pii/S0950705124011110). Cheapest fix protecting everything downstream.

**#3 — Deflated Sharpe / PBO against CUMULATIVE registry trial count.** (Effort: M)
BH-FDR corrects per-batch; nothing corrects for all-time multiplicity across brain.db. Best-of-N is spuriously significant as N grows (https://github.com/ml4t/diagnostic). Compute DSR using total trials-ever as N; add to deep_validation scorecard. Novelty gate handles duplicates; DSR handles multiplicity — both needed.

**#4 — Tag entry family (MR vs momentum) in DSL; split exit policy by family.** (Effort: M)
Kaminski & Lo: stops REDUCE expected return under random walk, add value only under momentum (https://papers.ssrn.com/sol3/papers.cfm?abstract_id=968338); mass-backtests confirm stops hurt mean-reversion, help trend (https://www.buildalpha.com/stop-losses-complete-guide/). MR methods: tighter timeouts, wider price stops, target-at-mean. Momentum methods: wider timeouts, ONLY family where trailing is worth a trial. Any exit study not conditioned on entry family averages two opposite effects. Add `family` tag to trials (also serves regime attribution, persistent_edges rec 7).

**#5 — Log MAE/MFE + fill-quality telemetry in shadow ledger NOW.** (Effort: S)
Two floats per trade (MAE/MFE) + retries/partial fills/quote staleness. At ≥80–100 forward trades per method, generate candidate barriers from winner-MAE 85–90th percentile / winner-MFE quantiles and submit as NEW TRIALS — never hand-edit live params (https://www.tradesviz.com/blog/mfe-mae-charts/, https://journalplus.co/learn/guides/mae-mfe-guide/). Infra failure modes dominate strategy failure modes in production (https://saintquant.com/blog/161-how-to-build-a-profitable-crypto-trading-bot-in-2026-a-quantitative-guide-for-algorithmic-traders). Cheap now, impossible retroactively.

**#6 — Harden shadow promotion: ≥30 → ≥50 trades + divergence/slippage bounds.** (Effort: S)
A 3–5pp win-rate estimation error (common below ~50 trades) inflates Kelly fraction 3× (https://en.wikipedia.org/wiki/Kelly_criterion). Raise forward_confirmed to ≥50; add numeric bounds: max divergence shadow-fills vs backtest-implied fills, live-vs-expected slippage cap. Candidates passing lockbox but failing shadow should be EXPECTED (Polymarket engine: 522× sim → −49.5% live, https://medium.com/@gwrx2005/ai-augmented-arbitrage-in-short-duration-prediction-markets-live-trading-analysis-of-polymarkets-8ce1b8c5f362; shadow-mode practice: https://alexgude.com/blog/machine-learning-deployment-shadow-mode/).

**#7 — Reweight DSL search toward 1d–1w momentum lookbacks on 15m bars.** (Effort: M)
The persistent horizon is daily/weekly, not 15m→15m: 1-week TSM Sharpe ~0.45 academic (https://acfr.aut.ac.nz/__data/assets/pdf_file/0009/918729/Time_Series_and_Cross_Sectional_Momentum_in_the_Cryptocurrency_Market_with_IA.pdf), trend-following Sharpe >1.5 in 2025 study (https://papers.ssrn.com/sol3/papers.cfm?abstract_id=5209907). Current features max out at ret20/range20/96-bar lookbacks (~24h). Add multi-day features (§2) and steer the LLM proposal prompt toward them. Pure 15m-lookback signals fight microstructure noise where evidence is thinnest.

**#8 — Cross-sectional rank frame.** (Effort: M–L)
Strongest ML-factor results are cross-sectional, not time-series (https://www.sciencedirect.com/science/article/abs/pii/S0927538X25003701, https://onlinelibrary.wiley.com/doi/full/10.1002/fut.22425). Rank each symbol's ret_1h/4h/24h, vol, funding across live universe per bar; expose as DSL primitives (`xrank_ret24 > 0.9`). Our frame is per-coin time-series only today — this is the biggest structural feature gap.

**#9 — Funding as mandatory context gate, never a carry sleeve.** (Effort: S)
`funding_z` exists as primitive; add: hard/flag gate blocking LONGs at extreme-positive funding z (crowding/crash signal — a 10% carry rise predicts ~22% more short liquidations, https://www.bis.org/publ/work1087.pdf), plus funding Δ (dynamics are forecastable, https://papers.ssrn.com/sol3/papers.cfm?abstract_id=5576424). Do NOT build delta-neutral carry: Sharpe collapsed 6.45 → 4.06 (2024) → negative (2025) (https://arxiv.org/pdf/2510.14435). Slots in as Layer 6 next to the existing counter-momentum gate.

**#10 — Kelly small-sample floor + momentum crash kill-switch.** (Effort: S)
Sizing already matches literature (×0.5 haircut ≈ McLean–Pontiff 58% post-publication decay https://arxiv.org/pdf/2209.13623; C_DD=0.5 = half-Kelly keeps ~75% growth, cuts P(half-bankroll) 1/2→1/8). Add: hard shrink-toward-zero when per-method n<50 (LCB z=1.5 at n<500 is not enough at n=30), and a volatility-scaled kill-switch on momentum-family sleeves — crypto momentum's tail is crash-prone (https://www.sciencedirect.com/science/article/abs/pii/S1544612325011377). Do NOT raise C_DD to 0.7 before live proof.

1. **Cross-sectional ranks** `xrank_ret1h/4h/24h`, `xrank_vol`, `xrank_funding` — universe-wide percentile per bar. Strongest literature support; directly feeds DSL comparators.
2. **Multi-day momentum**: `ret_1d`, `ret_3d`, `ret_7d`, `dist_7d_high_pct`, `dist_7d_low_pct` — moves search onto the persistent horizon (#7 above).
3. **Funding dynamics**: `funding_delta_8h` (8h-over-8h change), `funding_x_ch24` interaction (crowded-long into pump = crash risk). `funding_z` exists; make its window explicitly rolling-30d.
4. **Parkinson/Garman-Klass vol** over 24h of 15m bars + `gkvol_ratio_7d` — 5–7× more efficient than close-to-close, captures liquidation wicks ATR misses (https://arxiv.org/html/2404.04962v1); better normalizer for ATR-unit exits (#1) and gates.
5. **Taker-buy ratio z-score** (Binance kline field 10/field 8) — poor-man's OFI; predictive but regime-dependent, gate on vol regime (https://arxiv.org/html/2602.00776v1).
6. **Session flags**: `nyse_open_day` (0/1), `bars_to_funding` (0/8/16 UTC distance). `hour_utc`/`dow` already exist — keep for VOLATILITY conditioning only, not return prediction (https://www.sciencedirect.com/science/article/pii/S1059056024006506; https://quantpedia.com/the-seasonality-of-bitcoin/).
7. **`xrank_ret1h` tagged REVERSAL** — intraday perp reversal is liquidation-cascade-driven (https://www.sciencedirect.com/science/article/abs/pii/S154461232501058X) but is liquidity-provision premium: any winner must survive taker fees + realistic spread in bootstrap, flag if PnL concentrates in low-volume coins/hours (https://www.sciencedirect.com/science/article/pii/S0378426625000317).

Deferred (needs one extra endpoint, not OHLCV+funding): OI features — ΔOI_1h/24h normalized, 4-quadrant flag (price↑OI↑ new longs vs price↑OI↓ short-cover), OI-flush exhaustion. First in line once OI ingestion exists (https://insights.glassnode.com/leverage-position-openings-and-closures/).

Every feature enters as trials through novelty gate + lockbox, never as a default.

## 3) EXIT-ENGINEERING VERDICT on fixed sl/tp/timeout

**KEEP the structure — it IS the triple-barrier method (López de Prado), which is the literature's reference design.** Timeout as grid axis is already right (vertical barrier = cheapest dead-money killer; interacts with funding windows, so fund+fees stay in every exit comparison). Three changes, in order:
1. Units: raw % → ATR/σ multiples (improvement #1). The only structural flaw.
2. Condition by entry family (improvement #4). Stops on MR entries sell exactly before the reversion — testing trails on MR is a known dead end.
3. Barrier updates come from forward MAE/MFE quantiles at n≥80–100, submitted as new trials (improvement #5).

**Skip partial exits entirely**: expectancy-neutral at best, doubles fee events at 15m, and variance-smoothing is already handled by empirical Kelly (https://quantstrategy.io/blog/machine-learning-for-exit-optimization-predicting-the-best/, https://tradewiththepros.com/scaling-in-and-out-of-trades/). **Trailing**: momentum family only; test break-even ratchet after +1R alone first, then 1.5–3× ATR14 trail (https://www.luxalgo.com/blog/5-atr-stop-loss-strategies-for-risk-control/); the trailing benefit is a momentum-regime effect, not a free lunch (https://arxiv.org/pdf/1701.03960).

## 4) HYPE LIST — ignore

- **"Autonomous LLM trader beats market"** — only 2/77 studies had time-consistent splits, 1/77 explicit transaction costs (https://arxiv.org/html/2605.19337v1); most fail vs buy-and-hold (https://stockbench.github.io/); pretraining look-ahead contamination is real (https://arxiv.org/pdf/2601.13770).
- **Multi-agent debate frameworks as edge** (TradingAgents et al., https://arxiv.org/pdf/2412.20138) — untested vs cost-adjusted mechanical baselines. Note: `tradingagents_crypto_src/` sits in our repo — treat as reference code, not an edge source.
- **Delta-neutral funding carry sleeve / "19% APY" marketing** — academic Sharpe collapsed to negative in 2025.
- **Passive AMM LP** — structurally −EV via LVR, >$230M extracted from LPs (https://arxiv.org/pdf/2208.06046).
- **Weekend/day-of-week RETURN effects** — not robust 2014–2024; only vol/volume seasonality is real. Exception worth ONE registered trial: Monday-Asia-open trend (https://concretumgroup.com/seasonality-in-bitcoin-intraday-trend-trading/).
- **More TA indicators (RSI/MACD variants)** — 54–56% accuracy ceilings, mostly overfitting documentation (https://www.mdpi.com/2674-1032/4/4/77).
- **LSTM/deep sequence models** — XGBoost-class matches or beats on our sample sizes.
- **AI-predicted scale-out points, universal trailing %s from equity studies, "exit efficiency %" dashboards** — pure Goodhart.
- **Social-sentiment/on-chain feeds** — not in our data, weak intraday evidence.
- **Any backtest without explicit fee+slippage+funding model** — void by definition; this single omission invalidated 18/19 published LLM-trading studies.
